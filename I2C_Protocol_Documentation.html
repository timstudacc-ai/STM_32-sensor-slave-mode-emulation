Smart Sensor Emulator

I2C Protocol Implementation Documentation

Overview

The STM32F411 BlackPill acts as an I2C slave exposing an 8-byte register map. It emulates a hardware sensor by providing access to an internal temperature sensor, a user button counter, and an LED controller.

Slave Address: 0x42 (7-bit) -> 0x84 (Write), 0x85 (Read)

Register Map

Address Name Access Description
0x00 DEV_ID R Device identifier, hardcoded to 0xA5.
0x01 FW_VERSION R Firmware version, hardcoded to 0x01.
0x02 TEMP_RAW_H R Raw ADC value (12-bit) high byte. Updated every 500ms.
0x03 TEMP_RAW_L R Raw ADC value (12-bit) low byte.
0x04 TEMP_DEG_C R Calculated internal temperature in Celsius (Signed 8-bit).
0x05 BTN_COUNT R Increments on STM32 PA0 button press (debounced 50ms).
0x06 LED_CTRL R/W Bit 0 controls PC13 LED (1 = ON, 0 = OFF).
0x07 STATUS R Bit 0 = ADC Err, Bit 1 = I2C Err.

I2C Transaction Formats

The STM32 slave uses an auto-incrementing register pointer. This allows the master to read or write multiple sequential bytes efficiently (burst mode).

Overview

The Arduino Uno acts as the I2C Master, written in pure bare-metal C (`avr-libc`) without the Arduino Framework or `Wire.h`. It periodically polls the STM32 slave to read sensor data and sends commands to toggle the LED.

Bare-Metal TWI (I2C) Driver

The TWI driver operates at 100 kHz. It interacts directly with the ATmega328P hardware registers (`TWCR`, `TWSR`, `TWDR`, `TWBR`).

Key Features:

  • Timeout Protection: Every while loop checking the `TWINT` flag includes a millisecond timeout check using Timer0. This prevents the master from permanently freezing if a slave holds the clock line indefinitely.
  • Bus Recovery: If a timeout or bus error occurs, the master can re-initialize the bus by taking over the SDA/SCL pins as standard GPIOs and manually toggling the clock line 9 times to unstick the slave state machine.

API Usage Example

// Reading 3 bytes of Temperature data starting at register 0x02
uint8_t buf[3];
TWI_Status_t st = TWI_ReadRegister(0x42, 0x02, buf, 3);

if (st == TWI_OK) {
    uint16_t raw_temp = (buf[0] << 8) | buf[1];
    int8_t deg_c = (int8_t)buf[2];
    // Process temperature...
}

System Architecture

The master utilizes a non-blocking architecture using a Timer0-driven tick system.

  • Tick_Init() sets up Timer0 in CTC mode to fire an interrupt every 1ms, incrementing a global counter.
  • UART_Init() sets up UART0 at 115200 baud for debugging. Output can be viewed on a serial monitor.
  • The main super-loop uses Tick_GetMs() to execute the polling routine precisely every 1000ms without blocking CPU execution.
